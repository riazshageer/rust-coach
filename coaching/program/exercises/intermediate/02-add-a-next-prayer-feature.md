# Intermediate Exercise 02: Add A Next Prayer Feature

## Target Skill

Place new behavior in the layer where its meaning is strongest.

## Change Request

- Add a feature that identifies the next upcoming prayer for the current schedule.
- Decide whether the logic belongs in the domain, app, service, or formatter.
- Keep it testable and easy to explain.

## What The Coach Should Press On

- Does the chosen layer own the meaning?
- Is presentation staying separate from logic?
- Is the design solving today's problem without speculative expansion?

## Success Criteria

- the feature works
- the logic lives in the right place
- the design remains easy to test
