# Intermediate To Advanced

## What Progress Looks Like

- You can justify or reject a trait based on a real capability boundary.
- You can explain why a specific abstraction is worth its cost.
- You use commit history and tests as architectural evidence.
- You can reason about serialization, config, and CLI boundaries cleanly.
- You can discuss concurrency and async without reaching for them by reflex.

## Common Friction At This Stage

- overgeneralizing after one new requirement
- chasing elegant patterns that do not pay for themselves
- optimizing readability away in the name of performance or cleverness

## Coaching Focus

- sharpen tradeoff language
- strengthen API design
- raise the bar for abstraction
- connect code changes to system behavior and maintenance cost
