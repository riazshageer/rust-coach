# Session 17: Building Extensible Systems

## Outcome

Design for believable future change without turning the current code into a speculative platform.

## What This Session Is Really Teaching

Extensibility is about preserving good seams, not inventing plugin systems before there is pressure.

## Source Focus

- output formatting
- configuration and startup
- any trait boundaries you have introduced

## Work To Attempt

1. Name one change the app is likely to need.
2. Name one change it is unlikely to need soon.
3. Decide how the current design should prepare for the first but ignore the second.

## Exit Criteria

- you can design for likely change without speculative machinery
- you can explain why some future-proofing is waste
- you can preserve a clean seam where it matters
