# Session 01: Working Like A Rust Learner With A Coach

## Outcome

Set up the repo so each future session starts with context, a clear target, and a review loop.

## What This Session Is Really Teaching

Strong coaching is not passive. You need a repeatable way to resume work, state what you think is happening, attempt a change, and get precise feedback.

## Coaching Loop

1. Explain what the application does without using OOP vocabulary.
2. Update `coaching/state/current-session.md`.
3. Ask the coach to summarize the current architecture.
4. Identify one boundary you do not yet understand.

## What You Should Produce

- a short note in `current-session.md`
- your own description of the app's main data flow
- one concrete question for the coach

## Coach Expectations

The coach should reconstruct context from local files and git state, then slow you down if you ask for implementation before you can describe the problem.

## Exit Criteria

- you know how to start a session
- you know where persistent memory lives
- you can describe the app as data flowing across boundaries
