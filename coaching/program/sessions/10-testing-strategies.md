# Session 10: Testing Strategies

## Outcome

Treat tests as design pressure, not just bug nets.

## What This Session Is Really Teaching

The right test often proves the boundary is well-shaped. The wrong test often exposes that the design is too coupled or too vague.

## Source Focus

- domain constructors
- formatting behavior
- any new code you add in exercises

## Work To Attempt

1. Choose one behavior that matters architecturally.
2. Decide which level should test it.
3. Write down what a good test would prove beyond "it works."

## Exit Criteria

- you can explain why a test belongs at a specific layer
- you can distinguish behavior tests from architecture tests
- you can name one important case the current suite should eventually cover
