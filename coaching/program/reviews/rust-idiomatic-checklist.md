# Rust Idiomatic Checklist

- Are domain concepts represented by meaningful types?
- Are constructors enforcing important invariants?
- Is the API shape guiding callers toward correct use?
- Are enums or newtypes doing work that strings and flags should not?
- Is ownership explicit and reasonable?
- Are abstractions justified by the code in front of me?
- Does the code read like Rust, or like another language wearing Rust syntax?
